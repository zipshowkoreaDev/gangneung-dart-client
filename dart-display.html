DISPLAY

Dart Socket API – Display 문서

연결 안 됨

개요

Display는 게임 화면(관전자 뷰)입니다. 주 역할은 LISTEN — 컨트롤러가 보낸 조준·다트·게임 결과 이벤트를 받아 화면에 렌더링합니다. 제어용으로 EMIT 몇 가지를 발행할 수 있습니다.

연결 정보

URL:       https://socket-relay.zipshowkorea.com/dart
Transport: WebSocket / Polling (auto)
Library:   socket.io-client@4.x
Query:     ?room=<방 이름>   (필수 — 연결 시 해당 룸에 자동 참가)

기본 사용 예시

import io from 'socket.io-client';

const socket = io('https://socket-relay.zipshowkorea.com/dart', {
  query: { room: 'room-1' },
});

// 참가자 수/리스트 갱신
socket.on('joinedRoom', (d) => renderPlayers(d.players));

// 다트 조준 실시간 오버레이
socket.on('aim-update', (d) => renderAim(d.name, d.aim));
socket.on('aim-off',    (d) => clearAim(d.name));

// 다트 꽂힘
socket.on('dart-thrown', (d) => renderDart(d.name, d.aim, d.score));

// 게임 종료 화면
socket.on('game-finished', (d) => renderRanking(d.ranking));
Display는 observer로 동작할 수 있습니다. Controller와 달리 joinRoom을 호출하지 않으면 방의 참가자 수(playerCount)에 포함되지 않습니다. 연결 쿼리의 room 하나로 브로드캐스트만 받도록 구성하세요.

LISTEN Server → Display

Display의 핵심. 아래 이벤트가 들어오는 즉시 오른쪽 패널에 반영됩니다.

LISTENjoinedRoom

참가자가 들어오거나 나갈 때마다 방 전체에 발행. 참가자 리스트를 포함하므로 플레이어 네임태그 렌더링에 사용.

{
  room: string,
  playerCount: number,
  players: Array<{ socketId: string, name: string }>
}
LISTENroomPlayerCount

인원 수만 가볍게 받을 때. joinedRoom과 함께 발행.

{ room: string, playerCount: number }
LISTENaim-update

컨트롤러의 실시간 조준 좌표. 다트보드 위 조준선/커서 렌더링용.

{ room: string, name: string, aim: { x: number, y: number } }
LISTENaim-off

조준 종료. 커서 숨기기·조준선 제거에 사용.

{ room: string, name: string }
LISTENdart-thrown

다트가 꽂힌 순간. 보드에 다트를 그리고 점수를 표시.

{
  room: string,
  name: string,
  aim: { x: number, y: number },
  score: number
}
LISTENgame-result

게임 종료 결과 전체 패키지. 각 소켓별 개인 결과 + 전체 랭킹 포함.

{
  results: {
    [socketId]: {
      result: 'win' | 'lose' | 'tie',
      score: number,
      rank: number,
      totalPlayers: number,
      ranking: Array<{ name, score, rank }>
    }
  },
  ranking: Array<{ socketId, name, score, rank }>
}
LISTENgame-finished

디스플레이 전용 랭킹 요약. 화면 중앙 우승 배너/랭킹 보드 렌더링에 사용.

{ room: string, ranking: Array<{ name, score, rank }> }
LISTENstatus-queue

대기열(큐)이 바뀔 때마다 모든 클라이언트에게 발행. 대기 순번 UI에 사용.

string[]   // socketId 배열

EMIT Display → Server

Display가 세션을 제어하기 위해 보내는 이벤트. 실제 서버로 발행해볼 수 있습니다.

EMITdisconnect-room

해당 방의 모든 소켓(컨트롤러 + 디스플레이 + observer)을 강제 연결 해제. 게임 종료 후 GAME_END_CLOSE_DELAY_MS(5초) 대기 후 방 정리용으로 호출.

// payload
{ room: string }

socket.emit('disconnect-room', { room: 'room-1' });
실서버 영향 있음 — 방의 모든 소켓이 끊깁니다
EMITreset-queue

대기열 초기화. 서버는 project 룸으로 reset-queue를 브로드캐스트.

// payload
{ project: string }

socket.emit('reset-queue', { project: 'dart' });
EMITstatus-queue

현재 대기열 스냅샷을 요청. 응답은 요청자에게만 전달.

socket.emit('status-queue');
socket.on('status-queue', (queue) => renderQueue(queue));
