CONTROLLER

Dart Socket API – Controller 문서

연결 안 됨

개요

Controller는 플레이어가 다트를 던지고 조준하는 단말입니다. 서버의 dart 네임스페이스에 연결하여 EMIT 으로 액션을 보내고, LISTEN 으로 큐 상태와 게임 결과를 받습니다.

연결 정보

URL:       https://socket-relay.zipshowkorea.com/dart
Transport: WebSocket / Polling (auto)
Library:   socket.io-client@4.x
Query:     ?room=<방 이름>   (선택)

기본 사용 예시

import io from 'socket.io-client';

const socket = io('https://socket-relay.zipshowkorea.com/dart', {
  query: { room: 'room-1' },
});

socket.on('connect', () => {
  socket.emit('joinRoom', { room: 'room-1', name: 'Junn' });
});

socket.on('joinedRoom', (data) => console.log('joined', data));
권장 구조: Controller는 먼저 큐 단계에서 참가를 확정하고, 이후에만 게임 단계 이벤트를 보냅니다. 즉, join-queue / status-queue 로 로비를 처리하고, game-started 이후에 aim-update, throw-dart, aim-off 를 사용합니다.
제한: 방 이름 최대 64자, 이름 최대 32자, 방당 최대 4명. 초과 시 roomFull 이벤트가 해당 클라이언트에게만 발행됩니다.

EMIT Controller → Server

컨트롤러가 서버로 보내는 이벤트. 버튼을 눌러 실제 서버로 전송해볼 수 있습니다.

EMITjoinRoom

방에 참가. 이름·방 이름을 등록한다. 재접속·방 이동 시에도 호출.

// payload
{ room: string, name?: string }

socket.emit('joinRoom', { room: 'room-1', name: 'Junn' });
실시간 서버로 전송
권장: joinRoom은 큐 입장과 분리해서 사용합니다. 큐에 들어갈 때는 join-queue, 실제 게임 세션 참가가 확정된 뒤에만 게임용 등록을 수행합니다.
EMITaim-update

조준 좌표를 방 전체에 실시간 브로드캐스트. (좌표 범위는 프로젝트 규약)

// payload
{ room: string, name: string, aim: { x: number, y: number } }

socket.emit('aim-update', { room, name, aim: { x: 0.5, y: 0.3 } });
EMITaim-off

조준 종료 신호 (터치 떼기 등). 방 전체에 브로드캐스트.

// payload
{ room: string, name: string }

socket.emit('aim-off', { room, name });
EMITthrow-dart

실제 다트를 던진 순간. 최종 좌표와 점수를 포함해 방에 브로드캐스트.

// payload
{
  room: string,
  name: string,
  aim: { x: number, y: number },
  score: number
}

socket.emit('throw-dart', { room, name, aim: { x: 0.5, y: 0.3 }, score: 50 });
EMITfinish-game

게임 종료. 참가자 점수 리스트를 보내면 서버가 순위를 계산해 game-result / game-finished를 브로드캐스트.

// payload
{
  room: string,
  scores: Array<{ socketId: string, name: string, score: number }>
}

socket.emit('finish-game', {
  room: 'room-1',
  scores: [
    { socketId: 'abc', name: 'Junn',  score: 120 },
    { socketId: 'def', name: 'Jay',   score: 90 },
  ],
});
EMITjoin-queue

대기열(큐)에 자신의 소켓 ID 추가. 서버는 전역으로 status-queue를 브로드캐스트.

socket.emit('join-queue');   // no payload
EMITleave-queue

대기열에서 나간다.

socket.emit('leave-queue');  // no payload
EMITstatus-queue

현재 큐 상태를 요청한 클라이언트에게만 반환 (브로드캐스트 아님).

socket.emit('status-queue');
socket.on('status-queue', (queue) => console.log(queue));

LISTEN Server → Controller

아래 이벤트는 서버에서 자동으로 들어옵니다. 오른쪽 로그에서 실시간으로 확인하세요.

LISTENjoinedRoom

방에 참가자가 들어오거나 나갈 때마다 방 전체에 발행. 참가자 리스트 포함.

{
  room: string,
  playerCount: number,
  players: Array<{ socketId: string, name: string }>
}
LISTENroomPlayerCount

인원 수만 간단히 받을 때. joinedRoom과 함께 발행됨.

{ room: string, playerCount: number }
LISTENroomFull

방 정원(4명) 초과 시 해당 클라이언트에게만 발행. 참가 실패.

{ room: string, maxPlayers: number }
LISTENgame-result

finish-game 후 방 전체에 전송. 각 소켓ID별 개인 결과 + 전체 랭킹 포함. 클라이언트는 자신의 socketId 항목만 UI로 표시.

{
  results: {
    [socketId]: {
      result: 'win' | 'lose' | 'tie',
      score: number,
      rank: number,
      totalPlayers: number,
      ranking: Array<{ name: string, score: number, rank: number }>
    }
  },
  ranking: Array<{ socketId: string, name: string, score: number, rank: number }>
}
LISTENgame-finished

게임 종료 알림. 디스플레이용 랭킹 요약.

{ room: string, ranking: Array<{ name, score, rank }> }
LISTENgame-started

새 게임 세션 시작 알림. 이 이벤트를 받으면 로비 상태를 종료하고 해당 플레이어 배열 기준으로 턴 UI를 준비합니다.

{ room?: string, players?: string[] }
LISTENstatus-queue

현재 구현에서는 큐 상태 확인과 큐 변경 반영에 모두 사용합니다. 권장 해석은 로비 단계 전용 이벤트입니다.

string[]   // socketId 배열
